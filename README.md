## 1. Project Overview

**Product:** Bình Trà Tử Sa Long Phụng (Dragon Phoenix Purple Clay Teapot)

A Vietnamese e-commerce landing page selling a color-changing purple clay teapot.
The teapot changes color when hot water (>45°C) is poured in, making it a premium
tea accessory and gift item for the Vietnamese market.

**Business model:**
- Single product, direct-to-consumer sales
- Cash on delivery (COD) as primary payment method
- Bundle pricing to increase average order value
- Positioned as: personal use + luxury gift + feng shui item

**Pricing:**

| Bundle | Price | Notes |
|--------|-------|-------|
| 1 bộ (1 set) | 299,000₫ | (from 450,000₫ — 34% off) |
| 2 bộ (2 sets) | 579,000₫ | Save 19,000₫ vs 2× single |
| 3 bộ (3 sets) | 870,000₫ | Free shipping |

## 3. WordPress Architecture Strategy

### 3.1 What Lives in This Repository

This repository is **NOT** a full WordPress installation. It contains only the
**project-specific code** that you own and maintain:

```
This repo contains:
  wp-content/themes/flatsome-child/   ← your custom child theme
  wp-content/plugins/.gitkeep         ← placeholder (plugins installed via WP admin)
  wp-content/uploads/.gitkeep         ← placeholder (media uploaded via WP admin)
  database/import.sql                 ← WordPress + WooCommerce database setup
  deploy/                             ← deployment config templates
  .gitignore                          ← excludes WP core, parent theme, etc.
```

### 3.2 What Does NOT Live in This Repository

| Item | Reason | Where to get it |
|------|--------|-----------------|
| WordPress core (`wp-admin/`, `wp-includes/`) | Not owned by you, always download fresh | wordpress.org/download |
| Flatsome parent theme | Licensed commercial theme | See Flatsome theme source above |
| Plugins | Installed via WordPress admin | WordPress.org / plugin providers |
| `wp-config.php` | Contains database credentials (secrets) | Use `deploy/wp-config.php.example` as template |
| `wp-content/uploads/` | Media files, environment-specific | Upload manually or restore from backup |
| `.htaccess` | Server-specific, auto-generated by WordPress | Use `deploy/htaccess.example` as template |

### 3.3 Separation of Concerns

```
┌─────────────────────────────────────────────────────────────┐
│  WordPress Core      → download from wordpress.org          │
│  Flatsome Theme      → licensed from ThemeForest            │
│  Plugins             → installed in wp-admin                │
├─────────────────────────────────────────────────────────────┤
│  flatsome-child/     ← THIS REPO (your custom code)         │
│  database/import.sql ← THIS REPO (your data setup)         │
│  deploy/             ← THIS REPO (your config templates)   │
└─────────────────────────────────────────────────────────────┘
```

### 3.4 Architecture Rationale

- **Child theme only** — never modify the parent Flatsome theme. Parent theme
  updates will overwrite any direct edits.
- **WooCommerce** handles product, pricing, cart, checkout, and order management.
  No custom ordering system needed.
- **Flatsome UX Builder** for visual layout editing without touching code.
- **PHP template** (`page-landing.php`) for the conversion-focused landing page,
  giving full control over HTML structure and performance.
- **SQL import** delivers a pre-configured starting state so the site is not
  set up from scratch on every deployment.

## 5. Child Theme Structure

### 5.1 What Goes in the Child Theme

| File/Folder | Purpose |
|-------------|---------|
| `style.css` | Theme declaration header + CSS design tokens + global base styles |
| `functions.php` | PHP entry point — loads all /inc/ files, nothing else |
| `inc/enqueue.php` | Registers and enqueues all CSS and JS assets |
| `inc/woocommerce.php` | WooCommerce theme support, field customisation, currency settings, AJAX handler |
| `inc/bundles.php` | Bundle data source of truth, render function, shortcodes |
| `inc/landing.php` | Sticky CTA footer bar, FAQ JSON-LD schema, countdown meta tag, body classes |
| `inc/performance.php` | WordPress head cleanup, emoji removal, security headers, honeypot spam protection, login limiter |
| `page-landing.php` | Full-page template for the landing page — no sidebar, no distraction |
| `woocommerce/checkout/thankyou.php` | Custom thank-you page with Vietnamese next-steps UX |
| `assets/css/landing.css` | All section-level CSS: hero, USP cards, features, specs, video, bundles, testimonials, FAQ, contact |
| `assets/css/woocommerce.css` | Checkout form, cart, payment method, order summary overrides |
| `assets/js/countdown.js` | Cookie-based rolling countdown timer (pure JS, no jQuery dependency) |
| `assets/js/landing.js` | AJAX bundle add-to-cart, sticky CTA show/hide, FAQ accordion, scroll animations |

### 5.2 What Stays in Flatsome / UX Builder

| Item | Stay in |
|------|---------|
| Site header layout | Flatsome Customizer (Appearance → Customize → Header) |
| Site footer layout | Flatsome Customizer |
| Logo | Flatsome Customizer → Logo |
| Navigation menus | WordPress Appearance → Menus |
| Additional promotional blocks | Flatsome UX Builder (editable per-page) |
| Product image gallery | WooCommerce product admin |
| Blog posts / category pages | Not used (single-product site) |

### 5.3 CSS Architecture

All design values are defined as CSS custom properties in `style.css` `:root {}`.
Section-specific styles use these tokens. This means:
- Changing the brand color = change 1 variable
- Typography adjustments = change 2 variables
- Spacing scale = change 4 variables

No inline styles on any reusable component.

## 6. WooCommerce Setup

### 6.1 Product Structure: Variable Product with Bundle Variations

**Chosen approach:** Single WooCommerce variable product with 3 price variations.

**Why this approach (not a custom order form):**
- Native WooCommerce checkout → proper order management in wp-admin
- COD payment works without custom code
- Orders are trackable, reportable, email-notifiable out of the box
- Easy to edit prices in WooCommerce → Products → Edit Variations
- Compatible with any WooCommerce extension (VNPay, etc.)
- Does not require an additional "order form" plugin

**Product:**
- Type: Variable product
- Slug: `binh-tra-tu-sa-long-phung`
- Attribute: `so_luong_bo` (Số Lượng Bộ)

**Variations:**

| Variation | Attribute value | SKU | Price |
|-----------|----------------|-----|-------|
| 101 | 1 bộ | BTTS-1BO | 299,000₫ |
| 102 | 2 bộ | BTTS-2BO | 579,000₫ |
| 103 | 3 bộ | BTTS-3BO | 870,000₫ |

### 6.2 Order Flow

```
Landing page
    │
    ▼
User clicks "Chọn Gói Này" on bundle card
    │
    ▼
AJAX → flatsome_add_bundle (clear cart, add variation)
    │
    ▼
Redirect → /thanh-toan/ (Checkout)
    │
    ▼
Simplified checkout form (Name, Phone, Address)
    │
    ▼
Payment: Cash on Delivery (COD)
    │
    ▼
WooCommerce order created → status: "Processing"
    │
    ▼
Custom thank-you page (Vietnamese next-steps)
    │
    ▼
Admin ships → marks order "Completed"
```

### 6.3 Shipping Configuration

| Rule | Setting |
|------|---------|
| Shipping to | Vietnam only |
| Standard shipping | 30,000₫ flat rate |
| Free shipping | Orders ≥ 870,000₫ (3-bundle) |

### 6.4 Currency

## 7. Flatsome UX Builder Strategy

### 7.1 What UX Builder Is Used For

The main landing page (`page-landing.php`) is a **PHP template**, not a UX Builder page.
This provides full control over HTML, performance, and accessibility.

UX Builder is used for:
- Additional promotional banners
- Reusable blocks (trust strip, urgency bar, footer CTA)
- Header/footer layout via Flatsome Customizer

### 7.2 What UX Builder Should NOT Control

| Item | Reason |
|------|--------|
| Bundle pricing cards | PHP-driven, synced with WooCommerce variation data |
| FAQ content | Must match JSON-LD schema output in `inc/landing.php` |
| Product specs table | Data-driven |
| Sticky CTA bar | JavaScript behavior |

### 7.3 Content Editable by Non-Technical Admin

| Content | Where to edit |
|---------|--------------|
| Product price | WooCommerce → Products → Variations |
| Product images | WooCommerce → Products → Product Gallery |
| Product description | WooCommerce → Products → Long Description |

## 8. Database / SQL Import

### 8.1 What the SQL Export Contains

See `database/import.sql`. Includes: site options, WooCommerce configuration,
WooCommerce pages, product + 3 bundle variations, product categories, Yoast SEO meta,
admin user, navigation menu registrations.

### 8.2 What Cannot Be Pre-Generated

| Item | How to handle |
|------|---------------|
| Flatsome UX Builder page JSON | Build in wp-admin → Pages → UX Builder after deploy |
| Product images | Upload via Media Library, assign to product |
| Plugin-generated tables | Auto-created on plugin activation |
| WooCommerce shipping zones | Set up in WooCommerce → Shipping after deploy |

### 8.3 Domain Replacement (run after import)

```sql
UPDATE wp_options SET option_value = 'https://yourdomain.com' WHERE option_name = 'siteurl';
UPDATE wp_options SET option_value = 'https://yourdomain.com' WHERE option_name = 'home';
```

For serialized data:
```bash
wp search-replace 'https://YOURDOMAIN.COM' 'https://yourdomain.com' --all-tables
```

See `database/README.md` for full instructions.

## 10. DirectAdmin Deployment Guide

See `deploy/deploy-checklist.md` for the full step-by-step checklist.

**Quick summary:**

```
1. DirectAdmin → Create MySQL DB + user
2. phpMyAdmin → Import database/import.sql
3. phpMyAdmin → SQL → Run domain replacement queries
4. Upload WordPress core to public_html/
5. Upload Flatsome parent theme → wp-content/themes/flatsome/
6. Upload flatsome-child/ → wp-content/themes/flatsome-child/
7. Copy deploy/wp-config.php.example → wp-config.php (fill in DB + salts)
8. Copy deploy/htaccess.example → .htaccess
9. wp-admin → Install + activate plugins
10. wp-admin → Themes → Activate "Flatsome Child"
11. wp-admin → WooCommerce → Shipping → Add Vietnam zone
12. wp-admin → Settings → Permalinks → Save
13. Upload product images, assign to product
14. Test order end-to-end
15. Change admin password
16. Set up SSL + HTTPS redirect
```

**Server requirements:** PHP 8.1+, MySQL 8.0+, 256M memory, 64M upload max.

## 11. Local Development Setup

### Using LocalWP (recommended)

1. Download LocalWP: https://localwp.com/
2. Create new site with PHP 8.1, MySQL 8.0
3. Copy `wp-content/themes/flatsome-child/` into the LocalWP site's themes folder
4. Add Flatsome parent theme to themes folder
5. Import `database/import.sql` via LocalWP's phpMyAdmin
6. Run domain replacement for `http://yoursite.local`
7. Activate child theme, install plugins

### Using WP-CLI

```bash
wp db import database/import.sql
wp search-replace 'https://YOURDOMAIN.COM' 'http://localhost:8080' --all-tables
wp theme activate flatsome-child
wp plugin install woocommerce yoast-seo wp-super-cache --activate
wp rewrite flush
```

## 13. Manual Steps Required After Deployment

### Critical (before going live)

- [ ] Change admin password (default: `admin123`)
- [ ] Upload product images → Media Library → assign to WooCommerce product
- [ ] Configure WooCommerce shipping zones (Vietnam, flat rate + free shipping)
- [ ] Flush permalinks: Settings → Permalinks → Save
- [ ] Install and activate all required plugins
- [ ] Update variation IDs in `inc/bundles.php` (check actual IDs after import)
- [ ] Test complete order flow end-to-end

### Important (before first real order)

- [ ] Install WP Mail SMTP — configure SMTP for order confirmation emails
- [ ] Update admin email from placeholder to real email
- [ ] Install UpdraftPlus — configure automated backups
- [ ] Verify SSL certificate is valid, enable HTTPS redirect in `.htaccess`

### Post-Launch

- [ ] Submit sitemap to Google Search Console
- [ ] Set up Google Analytics / Facebook Pixel

## 14. Handoff Notes

### For the Next Developer

1. **All custom code is in `wp-content/themes/flatsome-child/`** — never touch Flatsome parent theme files
2. **Bundle data is single-sourced in `inc/bundles.php`** — update prices there AND in WooCommerce product variations
3. **The landing page is `page-landing.php`** — not a UX Builder page. Edit the PHP file for structural changes.
4. **CSS design tokens are in `style.css` `:root`** — change colors/typography there first
5. **FAQ questions appear in both `page-landing.php` and `inc/landing.php`** — keep them in sync
   (recommended future improvement: move FAQ to a CPT for admin editability)

### Known Limitations

1. **Product variation IDs in `inc/bundles.php` default to 0** — update after import with real variation IDs
2. **FAQ and testimonials are hardcoded in PHP** — non-technical admins cannot edit without a developer
3. **No analytics tracking pre-configured** — add GA4/GTM after launch
4. **Flatsome UX Builder additions are not in git** — if you use UX Builder extensively, export layout JSON periodically
5. **Email delivery requires WP Mail SMTP** — install before going live
